使用 pthread 时遇到的资源不足的问题

Posted on 2019-06-20 星期四 in Programming • Tagged with pthread

项目里遇到了一个 pthread_create 调用失败、返回资源不足的问题。

走读了代码，发现代码里反复调用了 pthread_create 创建了很多短任务；但主线程里并没有等待，而是任由其运行到退出。

查阅了 pthread 手册，对于新创建的 thread，它必须被 join 或者被 detach，这样才能避免 thread 相关的内部数据结构的泄漏。

针对项目实际情况，因为不需要 join，所以在调用 pthread_create 创建线程之后，添加了 pthread_detach 的调用，解决资源泄漏的问题。

gettid() 编译问题

Posted on 2018-09-17 星期一 in Programming • Tagged with pthread

代码里用到的 gettid()，编译时有报错，说找不到对应的头文件。

根据 man gettid 里的说法，libc 并没有提供这个系统调用的包装：

NOTES
    Glibc does not provide a wrapper for this system call; call it using syscall(2).

    ...

所以在实际使用时，需要添加以下声明：

#include <sys/syscall.h>
#include <unistd.h>

#define gettid() syscall(SYS_gettid)

使用 pthread tls destructor 时遇到的问题

Posted on 2011-07-06 星期三 in Programming • Tagged with pthread

使用 pthread 线程时，可以通过 pthread_key_create() 这个 API，给新创建的线程添加一个回调函数，在线程退出时会被系统调用到，通常用于线程相关资源的清理，等等。

int pthread_key_create(
    pthread_key_t *key, void (*destructor)(void*));

特别要注意的是，在 destructor 被回调时，其参数肯定不为空，但此时 对应的 tls 已经被清空！ 所以，如果 destructor 里有依赖 tls 的，其运行结果肯定跟预期的不一样。

写代码的时候没注意，出了问题也没有头绪，仔细看了 pthread_key_create 的文档 才清楚：

An optional destructor function may be associated with each key …
